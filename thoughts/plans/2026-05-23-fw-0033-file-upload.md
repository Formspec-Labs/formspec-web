## Decision audit trail (delta against the design doc)

- **No new context** added to `CompositionProvider.tsx` beyond what's required for the field-component override to reach `attachmentStore`. If the existing prop-drilling already reaches the override, no shim is added.
- **Stub URI scheme** uses `attachment:demo-<counter>` (counter monotonically increasing per stub instance) so the URI value is recognizable in fixtures + tests without leaking adapter internals.
- **Hash algorithm** is SHA-256 (WebCrypto subtle.digest) with the same fallback path as `responseHash` in `respondent-flow.ts` for environments without subtle crypto.
- **AttachmentRef.kind** is the literal string `'attachment-ref'` — a discriminator so server-side / verifier-side detection of an `AttachmentRef` inside response data is keyed off a non-empty discriminator field, not a heuristic.
